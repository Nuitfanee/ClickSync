# Click Sync

> 基于 Web HID API 的跨品牌鼠标配置工具

Click Sync 是一个基于 Web HID API 的纯前端鼠标配置应用，旨在构建一个 **ALL IN ONE** 的网页驱动解决方案，支持多种品牌的鼠标设备。无需安装任何驱动或软件，直接在浏览器中完成鼠标的所有配置。

> 💡 **开发说明**：本项目在开发过程中使用了 AI 辅助开发工具（如 Gemini 3 Pro）进行代码构建。

---

## ✨ 功能特性

### 🎮 核心功能

- **按键映射**：自定义鼠标按键功能，支持多种按键映射选项，满足个性化需求
- **DPI 设置**：灵活配置多个 DPI 档位（1-6档），自定义 DPI 范围和每个档位的具体数值
- **基础性能**：
  - 回报率调整：支持 125Hz、250Hz、500Hz、1000Hz、2000Hz、4000Hz、8000Hz
  - 性能模式：低功耗（Low Power）、标准（Standard）、竞技（Competitive）、超频（Overclock）
- **高级参数**：
  - **电源管理**：休眠时间设置、按键防抖配置
  - **传感器调校**：角度修正、手感值调整
  - **高级性能开关**：
    - 移动同步（Motion Sync）：平滑算法修正，移动轨迹更准确
    - 直线修正（Angle Snap）：修正直线移动时的偏移抖动
    - 纹波修正（Ripple Fix）：消除高移速情况下的波浪形抖动
    - 辅助压枪（Recoil Assist）：自动识别并轻微滤除鼠标波动
  - **LED 指示灯控制**：DPI 档位颜色指示、接收器灯效设置
  - **LOD 静默高度**：设置传感器在鼠标抬起时的最小静默高度
  - **其他专属功能**：不同品牌设备可能拥有独特的配置选项

### 🧪 测试工具

- **双击检测**：实时检测鼠标按键的双击问题，统计按下/抬起次数、时间间隔，帮助诊断按键故障
- **轮询率检测**：实时检测鼠标回报率，显示峰值、平均值、抖动等关键指标，验证设备性能
- **灵敏度匹配**：智能算法帮助在不同鼠标之间匹配相同的移动灵敏度，实现无缝切换
- **鼠标角度校准**：自动检测并校准鼠标的握持角度，优化使用体验

## 📖 使用指南

### 1. 打开应用

直接在浏览器中打开 `index.html` 文件，或通过 Web 服务器访问。

> 💡 **推荐**：使用本地 Web 服务器（如 Python 的 `http.server` 或 Node.js 的 `http-server`）可以获得更好的体验。

### 2. 连接设备

1. 点击起始页的触发区域（"Hold to Initiate System"）
2. 浏览器会弹出设备选择对话框
3. 在列表中选择你的鼠标设备并点击"连接"

> ⚠️ **注意**：首次连接时，浏览器可能会要求授权访问 HID 设备，请点击"允许"。

### 3. 配置设备

连接成功后，可以通过左侧导航菜单访问各个功能模块：

- **按键设置**：点击鼠标图片上的按键点，在弹出的映射列表中选择要映射的功能
- **DPI 设置**：配置 DPI 档位数量（1-6档）和范围，设置每个档位的具体 DPI 值
- **基础性能**：选择性能模式（低功耗/标准/竞技/超频）和回报率（125Hz - 8000Hz）
- **高级参数**：调整传感器参数、高级性能开关、LED 指示灯等高级功能
- **测试工具**：使用双击检测、轮询率检测、灵敏度匹配、角度校准等工具

## ⚠️ 注意事项

### 重要提示

- ✅ **浏览器兼容性**：本应用需要浏览器支持 Web HID API，目前仅 Chrome/Edge 89+ 支持
- ✅ **参数理解**：修改设备参数前请确保了解各项设置的含义，避免误操作
- ⚠️ **游戏合规性**：某些高级功能（如辅助压枪）可能在某些游戏中被视为违规，请谨慎使用
- 💾 **配置备份**：建议在使用前记录或备份设备的原始配置，以便需要时恢复

### 测试工具使用建议

- **轮询率检测**：
  - 建议关闭浏览器的鼠标手势功能
  - 测试时只连接一个鼠标设备
  - 关闭 Windows 的"增强指针精确度"选项
- **灵敏度匹配**：
  - 关闭 Windows 的"增强指针精确度"选项
  - 关闭驱动的移动同步、直线修正、纹波修正等功能
  - 建议使用物理导轨进行测试，确保测试准确性
- **角度校准**：
  - 使用自然握姿进行测试
  - 保持水平滑动，避免垂直移动
  - 建议进行多次测试以获得更准确的结果

## 🔧 开发说明

### 设备适配器架构

项目采用**适配器模式**，实现了优雅的多设备支持：

- **协议隔离**：每个设备品牌拥有独立的协议实现文件（`protocol_api_*.js`）
- **统一管理**：通过 `DeviceAdapters` 注册表统一管理所有设备适配器
- **自动识别**：设备自动识别，根据 VID（Vendor ID）和 Usage Page 匹配对应协议
- **易于扩展**：新增设备品牌只需实现对应的协议适配器即可

### UI 变体系统

- **设备特定 UI**：不同设备可以拥有独立的 UI 样式和配置
- **动态应用**：通过 `DeviceUI.applyVariant()` 动态应用设备特定的 UI 变体
- **家族兼容**：支持设备家族（如 Rapoo 家族）的兼容性处理

### 协议实现架构

项目采用分层架构设计：

- **传输层**：`UniversalHidDriver` 负责设备通信，不关心业务逻辑
- **编码层**：`ProtocolCodec` 统一生成读写命令
- **配置层**：`DEFAULT_PROFILE` 定义机型能力和可用组合
- **转换层**：`TRANSFORMERS` 处理语义值与协议值的转换
- **规范层**：`SPEC` 定义语义配置项规范（验证/编码/计划/依赖/优先级）
- **计划层**：`CommandPlanner` 将配置变更转换为可执行命令
- **业务层**：`MouseMouseHidApi` 提供对外 API

这种架构使得新增功能时，通常只需要修改配置层，业务层无需改动。
